Metodología

El presente proyecto corresponde a una investigación de tipo aplicada, ya que tiene como finalidad desarrollar una solución tecnológica que permita mejorar la gestión del conocimiento en la lectura digital. Asimismo, se adopta un enfoque mixto, debido a que se combinan elementos cuantitativos y cualitativos. Por un lado, se analizarán datos relacionados con los hábitos de lectura y el uso de herramientas digitales; por otro, se buscará comprender la experiencia de los usuarios respecto a las dificultades que presentan al recordar y organizar la información que leen.

Para la recolección de datos se emplearán diversas técnicas. En primer lugar, se aplicarán encuestas a estudiantes y lectores digitales con el objetivo de identificar sus hábitos de lectura, las herramientas que utilizan, además de los principales problemas que enfrentan al intentar recordar lo aprendido. También se realizarán entrevistas que permitirán profundizar en la experiencia del usuario, obteniendo una visión más detallada sobre la forma en que gestionan su información. De igual manera, se llevará a cabo una revisión documental de fuentes académicas relacionadas con la memoria, la lectura digital, así como la gestión del conocimiento, con el fin de sustentar teóricamente el desarrollo del proyecto.

En cuanto a los materiales e instrumentos, se utilizarán herramientas digitales como Google Forms para la aplicación de encuestas, junto con guías estructuradas para las entrevistas. Para el desarrollo del sistema se empleará una computadora personal, el entorno de desarrollo Visual Studio Code, el lenguaje de programación Java con el framework Spring Boot, una base de datos relacional como MySQL o PostgreSQL, además de una API externa que permitirá la búsqueda de libros.

El diseño del sistema se desarrollará bajo un enfoque de ingeniería iterativo, basado en la construcción de un producto mínimo viable (MVP). Este proceso incluirá las etapas de análisis, en la que se identificarán las necesidades del usuario junto con los requerimientos del sistema; diseño, donde se definirá la arquitectura cliente-servidor y la estructura de la base de datos; implementación, en la que se desarrollará el backend e integrará la API externa; finalmente, la fase de pruebas, donde se verificará el correcto funcionamiento de las funcionalidades principales del sistema.

Finalmente, la validación del sistema se realizará comprobando que el usuario pueda buscar libros mediante la API, guardar sus libros favoritos, registrar resúmenes u opiniones personales, además de asegurar que toda la información se almacene correctamente en la base de datos. También se evaluará que el sistema sea funcional, además de fácil de utilizar para el usuario.
